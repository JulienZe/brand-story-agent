# 企业级前端界面设计需求规范

## 一、视觉风格规范

### 1.1 整体定位
- **风格调性**：专业、简洁、高效、可信赖
- **视觉隐喻**：采用「工具感」设计语言，界面元素应像精密仪器一样精准、可靠
- **去装饰化**：避免过度使用渐变、阴影、动画等装饰性元素
- **克制度**：视觉元素服务于功能，不为装饰而装饰

### 1.2 色彩系统

#### 主色调
| 用途 | 色值 | 说明 |
|------|------|------|
| 品牌主色 | #1A73E8 | 代表信任、科技、专业 |
| 品牌辅助色 | #34A853 | 成功状态、正向操作 |
| 警示色 | #EA4335 | 错误、危险、删除等高风险操作 |
| 警告色 | #FBBC04 | 警告、提示、注意 |
| 信息色 | #4285F4 | 信息提示、链接 |

#### 中性色
| 用途 | 色值 | 说明 |
|------|------|------|
| 主文本 | #202124 | 主要文字内容 |
| 次要文本 | #5F6368 | 辅助说明文字 |
| 占位文本 | #9AA0A6 | 输入框提示、禁用状态 |
| 分割线 | #DADCE0 | 分隔线、边框 |
| 背景色 | #FFFFFF | 主背景 |
| 页面背景 | #F8F9FA | 页面底色、卡片背景 |

#### 状态色
| 状态 | 色值 | 使用场景 |
|------|------|----------|
| 成功 | #34A853 | 操作成功、完成状态 |
| 警告 | #FBBC04 | 警告提示 |

### 1.3 字体系统

#### 字体族
```
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
```

#### 字号层级
| 用途 | 字号 | 字重 | 行高 |
|------|------|------|------|
| 页面标题 | 24px | 600 | 1.3 |
| 区块标题 | 18px | 600 | 1.4 |
| 卡片标题 | 16px | 500 | 1.5 |
| 正文 | 14px | 400 | 1.6 |
| 辅助文字 | 12px | 400 | 1.5 |
| 最小文字 | 11px | 400 | 1.4 |

### 1.4 间距系统
- **基础单位**：4px
- **间距层级**：4px / 8px / 12px / 16px / 24px / 32px / 48px / 64px
- **组件内间距**：通常使用 12px 或 16px
- **区块间距**：通常使用 24px 或 32px
- **页面边距**：桌面端 24px-48px，移动端 16px

### 1.5 圆角系统
| 元素类型 | 圆角值 |
|----------|--------|
| 按钮、输入框 | 4px |
| 卡片、面板 | 8px |
| 模态框、下拉面板 | 12px |
| 头像、标签 | 50%（圆形）/ 4px |

### 1.6 阴影系统
- **层次一**（轻微）：`0 1px 2px rgba(0,0,0,0.05)`
- **层次二**（标准）：`0 1px 4px rgba(0,0,0,0.1)`
- **层次三**（悬浮）：`0 4px 8px rgba(0,0,0,0.1)`
- **层次四**（弹出）：`0 8px 24px rgba(0,0,0,0.15)`

## 二、交互体验规范

### 2.1 按钮设计

#### 按钮类型
| 类型 | 样式 | 使用场景 |
|------|------|----------|
| 主按钮 | 实心蓝色背景 + 白色文字 | 主要操作、唯一主操作 |
| 次按钮 | 蓝色边框 + 蓝色文字 | 次要操作、辅助操作 |
| 文字按钮 | 无边框、无背景 | 轻微操作、辅助链接 |
| 危险按钮 | 实心红色背景 + 白色文字 | 删除、撤销等危险操作 |

#### 按钮规格
- **高度**：32px（小）/ 36px（中）/ 40px（大）
- **内边距**：左右 16px
- **字号**：14px，字重 500
- **禁用态**：透明度 50%，cursor: not-allowed

#### 交互状态
- **Hover**：背景色加深 10%
- **Active**：背景色加深 15%
- **Focus**：2px 蓝色 outline，offset 2px
- **Loading**：显示 spinner，文字变为"处理中..."

### 2.2 表单设计

#### 输入框规格
- **高度**：36px
- **内边距**：左右 12px
- **边框**：1px solid #DADCE0
- **圆角**：4px
- **字号**：14px

#### 交互状态
- **Normal**：灰色边框
- **Focus**：蓝色边框（#1A73E8），添加浅蓝色 box-shadow
- **Error**：红色边框（#EA4335），下方显示错误提示
- **Disabled**：灰色背景（#F1F3F4），光标禁止

#### 验证反馈
- **错误提示**：红色文字（#EA4335），字号 12px，位于输入框下方
- **成功状态**：绿色勾选图标
- **字符计数**：右对齐，灰色文字，超限时变红

### 2.3 导航设计

#### 顶部导航栏
- **高度**：56px
- **背景**：白色 + 底部 1px 分割线
- **Logo**：左侧，高度 24px-32px
- **导航项**：水平排列，间距 24px，当前项下方蓝色指示条
- **用户区**：右侧，头像 + 下拉菜单

#### 侧边导航（可选）
- **宽度**：240px（展开）/ 64px（折叠）
- **背景**：#F8F9FA
- **选中态**：左侧 3px 蓝色边框 + 浅蓝背景

### 2.4 加载状态

#### 全局加载
- 使用进度条（Linear Progress）而非 spinner
- 颜色：品牌蓝色
- 位置：页面顶部或操作区域内部

#### 局部加载
- 按钮内嵌 spinner
- 卡片内嵌骨架屏（Skeleton）
- 列表项骨架屏占位

#### 加载反馈原则
- 操作开始立即显示加载态
- 加载失败显示错误态 + 重试按钮
- 加载成功短暂显示成功态（300ms）后消失

### 2.5 空状态

#### 设计要点
- 使用简洁的线性插图（SVG）
- 主标题 + 辅助说明文字
- 提供引导性操作按钮
- 示例：
  - 图标：文件夹 + 放大镜
  - 标题：「暂无数据」
  - 说明：「当前没有符合条件的内容」
  - 操作：「创建第一个项目」按钮

### 2.6 错误处理

#### 表单错误
- 单字段错误：输入框下方红色提示
- 多字段错误：顶部通用错误提示 + 字段级错误
- 错误输入框：红色边框 + 浅红背景

#### 页面级错误
- 使用错误插图
- 清晰的中文错误说明
- 提供「返回首页」或「重试」按钮

#### 网络错误
- 检测网络状态
- 断网时显示离线提示横幅
- 请求失败显示「网络连接失败，点击重试」

## 三、功能模块规范

### 3.1 布局结构

#### 页面布局模式
```
┌─────────────────────────────────────┐
│           Header (56px)              │
├─────────┬───────────────────────────┤
│         │                           │
│  Side   │        Main Content       │
│  Nav    │                           │
│ (240px) │                           │
│         │                           │
│         │                           │
└─────────┴───────────────────────────┘
```

#### 内容区域
- 最大宽度：1200px（数据展示）/ 800px（表单录入）
- 居中对齐，两侧自适应留白
- 移动端全宽布局

### 3.2 通用组件

#### 卡片组件
- 背景：白色
- 边框：1px solid #DADCE0
- 圆角：8px
- 内边距：16px-24px
- 标题区 + 内容区 + 操作区

#### 表格组件
- 表头：#F8F9FA 背景，字重 500
- 行高：48px
- 斑马纹：奇偶行交替背景色
- 悬停态：浅蓝背景 #E8F0FE
- 选中态：更深蓝色 #D2E3FC
- 支持排序、分页、筛选

#### 模态框
- 居中显示
- 遮罩层：黑色 50% 透明度
- 最大宽度：480px（小）/ 640px（中）/ 800px（大）
- 标题区 + 内容区 + 操作区
- 支持 ESC 关闭、可选拖拽

#### 消息提示
- Toast 提示：从顶部居中滑入
- 自动消失：3秒（信息）/ 5秒（警告）
- 手动关闭：显示关闭按钮
- 类型：成功（绿）/ 警告（黄）/ 错误（红）/ 信息（蓝）

### 3.3 响应式断点

| 设备 | 宽度范围 | 布局调整 |
|------|----------|----------|
| 超小屏 | < 576px | 单列布局，导航折叠 |
| 小屏 | 576-768px | 紧凑布局，侧边栏收起 |
| 中屏 | 768-992px | 标准布局，侧边栏可选 |
| 大屏 | 992-1200px | 宽松布局，完整侧边栏 |
| 超大屏 | > 1200px | 最大宽度限制 |

## 四、技术规范

### 4.1 代码质量

#### 命名规范
- **CSS 类名**：小写 + 连字符（kebab-case），如 `.btn-primary`
- **JavaScript 变量**：小驼峰（camelCase）
- **React 组件**：大驼峰（PascalCase）
- **常量**：全大写 + 下划线

#### 代码风格
- 缩进：2 空格（HTML/JSX）/ 4 空格（JavaScript）
- 引号：双引号（HTML 属性）/ 单引号（JavaScript）
- 尾逗号：ES5 风格（最后一个属性后不加分隔）
- 分号：必须使用

### 4.2 性能指标

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间（LCP） | < 2.5s |
| 交互准备时间（TTI） | < 3.8s |
| 页面帧率（FPS） | ≥ 55fps |
| 最大包体积 | < 500KB（首屏） |
| HTTP 请求数 | < 20 个 |

### 4.3 可访问性（a11y）

- 所有交互元素可键盘访问
- 表单标签正确关联
- 图片提供 alt 文本
- 颜色对比度 ≥ 4.5:1
- Focus 状态清晰可见
- 支持屏幕阅读器

### 4.4 浏览器兼容

| 浏览器 | 最低版本 |
|--------|----------|
| Chrome | 最近 2 个版本 |
| Firefox | 最近 2 个版本 |
| Safari | 最近 2 个版本 |
| Edge | 最近 2 个版本 |
| iOS Safari | 最近 2 个版本 |
| Chrome for Android | 最近 2 个版本 |

## 五、品牌一致性

### 5.1 Logo 使用规范
- 主 Logo 尺寸：高度 32px，宽度自适应
- 最小可识别尺寸：高度 24px
- 与文字间距：≥ 8px
- 背景要求：浅色背景使用深色 Logo，深色背景使用浅色 Logo

### 5.2 产品语调
- **正面**：专业、简洁、温暖
- **避免**：过于技术化、学术化、生硬
- **示例**：
  - ✅「创建成功」
  - ❌「操作已成功执行」
  - ✅「请输入手机号」
  - ❌「手机号字段为必填项」

### 5.3 错误消息
- 使用用户能理解的语言
- 说明发生了什么 + 用户应该怎么做
- 示例：
  - ✅「用户名已存在，请尝试其他名称」
  - ❌「错误码：E1001，用户名唯一性校验失败」
